# Сервисные навыки

Все интеграции поставляются вместе с репозиторием и настраиваются по первому запросу. Они не содержат аккаунтов, cookies, токенов, персональных идентификаторов и браузерных профилей автора шаблона.

## Проверка

Показать доступные интеграции:

```bash
python3 scripts/integration_preflight.py --list
```

Проверить конкретную интеграцию:

```bash
python3 scripts/integration_preflight.py ozon-buyer-search
```

Код `0` означает готовую настройку, `2` – незавершённую первичную настройку репозитория, `3` – интеграция доступна, но ещё не включена владельцем.

## Общие границы

- Первый вход и проверка личности выполняются владельцем.
- CAPTCHA, SMS, push, TOTP, электронная подпись и восстановление доступа не автоматизируются.
- Навык может читать только выбранный пользователем кабинет, чат, заказ, обращение или диапазон операций.
- Финальная отправка, платёж, подпись или изменение аккаунта требуют отдельного разрешения либо остаются ручным действием по правилам конкретного навыка.
- Секреты не записываются в `workspace.json`, `PROFILE.md`, task logs или Git.

## Доступные интеграции

| Навык | Возможности из коробки | Обязательное участие владельца |
| --- | --- | --- |
| `ozon-buyer-search` | поиск, сравнение, карточки, отзывы, чтение выбранного собственного заказа | вход для личных заказов, CAPTCHA |

## Т-Банк на macOS и Windows

Базовый переносимый режим использует отдельный браузерный профиль или уже авторизованную сессию, защищённую учётной записью ОС. На macOS рекомендуется FileVault, на Windows – BitLocker. Windows Credential Manager и macOS Keychain могут хранить только настройки, которые владелец осознанно решил сохранить; публичный навык сам не сохраняет пароль или TOTP.

Если шифрование диска не подтверждено, навык может работать только в ручной интерактивной сессии без отдельного сохранения банковских секретов.
